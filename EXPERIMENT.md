# Trellis Is an Experiment

This document exists to be explicit about what Trellis is and what it isn't.

## What Trellis Is

Trellis is a **personal R&D project** by Kyle Morrand. It's an experiment in building a persistent AI agent runtime using existing tools and open-source infrastructure. The goal is to learn — about agent architecture, context management, human-AI coordination, and the practical realities of running an always-on AI assistant.

Trellis prototypes architectural principles that Kyle is also developing professionally at [Mirror Factory](https://mirrorfactory.ai), where the team is building **Layers**, a context management platform for teams and agents. The concepts are related. The code is not.

## What Trellis Isn't

- **Not a product.** There is no roadmap, no release schedule, no support.
- **Not Mirror Factory IP.** Trellis is personal experimental code under MIT license. Mirror Factory's product (Layers) is separate.
- **Not production-ready.** This is R&D. Things will break. Approaches will change. That's the point.
- **Not a fork of OpenClaw.** Trellis is built from scratch, inspired by OpenClaw's architecture and other open-source agent frameworks. See the README for credits.

## The Relationship to Layers

Kyle is CEO of Mirror Factory. Mirror Factory is building Layers. Kyle is also building Trellis as a personal experiment. The relationship:

- **Learnings transfer.** Insights from building Trellis inform the design of Layers. "Here's what worked, here's what didn't" — that knowledge goes back to the team.
- **Code does not transfer.** Trellis code stays in this repo under MIT license. Layers code stays in Mirror Factory's private repos.
- **Trellis is customer zero.** Mirror Factory's thesis is that organizations need context management infrastructure. Kyle is testing that thesis on himself first.

This is the same relationship as a game designer building a prototype in their garage to test mechanics before committing to full production. The prototype validates ideas. The production build is a separate effort.

## Why Make This Public?

Three reasons:

1. **Learning in public.** The AI agent space is moving fast and most people are watching from the sidelines. Sharing the build process — what works, what doesn't, what's hard — helps others learn too.
2. **Accountability.** Public experiments get finished more often than private ones.
3. **Community.** The best way to find people working on similar problems is to show your work.

## For Investors

If you're reading this as part of due diligence on Mirror Factory: Trellis is Kyle's personal experiment, not a company project. It validates the Layers thesis using personal time and personal resources. Think of it as a founder who uses their own product before asking customers to — except the product doesn't exist yet, so the founder built a prototype to test the principles. That's Trellis.
