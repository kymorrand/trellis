# Build Log — March 21, 2026

## Phase 1 Complete: Ivy Has a Heartbeat

**What happened today:**

Implemented the core runtime features that turn Ivy from a reactive chatbot into a proactive agent. Four major additions:

### 1. Heartbeat Scheduler
The proactive half of the dual-loop architecture is now live. The heartbeat runs as an asyncio background task alongside the Discord bot, ticking every 60 seconds and checking what's due:

- **Every 30 min** — Checks `_ivy/inbox/` for new files, logs to journal
- **Midnight** — Nightly vault backup (git add/commit/push), journal rollover with activity summary, cost report with budget alerts
- **8:00 AM** — Morning brief posted to Discord: overnight activity, queue count, vault stats
- **6:00 PM** — End of day summary: messages processed, vault saves, API cost
- Budget monitoring: alerts on Discord when monthly API spend exceeds 75%

### 2. Systemd Service
Ivy now survives reboots. Created a systemd service file with auto-restart on crash (10s delay), security hardening (NoNewPrivileges, PrivateTmp), and an install script that handles everything.

### 3. Auto-Context Assembly
The biggest quality-of-life improvement. On every non-trivial message, Ivy now:
1. Extracts 2-3 keywords from Kyle's message (filtering 150+ stop words)
2. Searches the vault with those keywords
3. If relevant results found (above a relevance threshold), includes them as context

This means mentioning "Mirror Factory" automatically pulls in the MF project index. Mentioning "Bobby" pulls in Bobby's profile. No more "check the vault" — Ivy just knows.

### 4. /status Command
Kyle can type `/status` in Discord to get an immediate status report: uptime, heartbeat ticks, vault file count, API spend today and this month (with budget percentage), and today's activity stats.

**Test coverage:**

96 tests across 6 test files, all passing:
- `test_heartbeat.py` — 18 tests: scheduling, journal parsing, briefs, backup, cost tracking
- `test_context.py` — 23 tests: keyword extraction, search gating, auto-context assembly
- `test_vault.py` — 13 tests: search, read, save, formatting
- `test_journal.py` — 6 tests: journal creation, logging
- `test_router.py` — 18 tests: model classification, prefix handling
- `test_soul.py` — 8 tests: SOUL.md parsing, condensed loading

**Key decisions:**

- Heartbeat uses simple datetime checks rather than the `schedule` library — fewer dependencies, easier to test, and asyncio-native
- Auto-context uses keyword extraction with a large stop word list rather than embeddings — fast, predictable, no GPU needed, good enough for a personal vault
- Morning brief and EOD summary are intentionally simple for now — calendar integration and Linear integration come in Phase 2

**What's next (Phase 2):**

- Google Calendar integration for morning briefs and calendar-aware notification queuing
- Linear integration for task tracking in briefs
- Midday check-in (12 PM) and weekly review (Sunday 8 PM)
- Vault hygiene automation (orphaned files, broken links, missing tags)
- Context refresh (re-index changed files for better search)
