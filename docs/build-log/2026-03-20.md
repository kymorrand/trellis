# Build Log — March 20, 2026

## Day 1: Laying the Foundation

**What happened today:**

Started the Trellis project — the personal agent runtime that will eventually host Ivy, my professional AI assistant.

The day started with a brain dump about how I work, what's broken, and what I want to build. I rambled about my tools (GitHub, Obsidian, Claude Code, Linear), my devices (Samsung Galaxy, two Lenovo Legions, a ThinkPad, and a reMarkable 2), and my primary working style: deep read & write sessions fueled by voice-note rambles.

From there, we ran the whole thing through the Layers 4C cycle as a paper prototype:

- **Collect** — Captured every tool, workflow, pain point, goal, and open question from the ramble
- **Curate** — Structured it into: what's working, what's broken, what's missing, what's decided, and where the tensions are
- **Coordinate** — Built a full system architecture spec: repo structure, module design, vault layout, device ecosystem, security policy, communication plan
- **Compound** — Started building: created the trellis repo, scaffolded the full Python package with module stubs, wrote SOUL.md for Ivy, and kicked off a fresh Ubuntu install on Ivy's dedicated machine (Lenovo Legion Pro 7, i9-13900HX, 32GB RAM, RTX 4090 Mobile)

**Key decisions made:**

1. Build our own runtime from scratch (inspired by OpenClaw, not forked from it)
2. Trellis = the platform/engine, Ivy = the agent/character running on it
3. Trellis is explicitly an experiment, not a product (see EXPERIMENT.md)
4. Anthropic-first for cloud inference, with local Ollama for cost and privacy
5. Python 3.12+, no LangChain, no third-party plugins
6. Conservative security posture from day one
7. Everything is public and documented — building in public as education

**Architecture highlights:**

- Dual-loop design: reactive event processing + proactive heartbeat scheduler
- Game-design vocabulary for modules: senses, mind, hands, memory
- File-based everything: Markdown knowledge, JSON state, YAML config
- Obsidian vault as the knowledge base (proto-Context Library)
- SOUL.md as the agent personality (proto-Ditto professional context model)

**What's next:**

- Finish Ubuntu setup on Ivy's machine (install Ollama, Docker, Tailscale)
- Create the ivy-vault repo and scaffold the Obsidian vault structure
- Set up a private Discord server ("The Greenhouse") for Ivy communication
- Build the first working version of core/loop.py — a minimal agent that can respond to a message
- Write the first "Growing the Garden" blog post about Day 1

**Thought for the day:**

The hardest part wasn't the technical architecture — it was deciding to build from scratch instead of using OpenClaw. But as a game designer, I know the difference between installing someone else's engine and understanding what makes an engine tick. Trellis is the second kind. The learnings go straight back into Layers.
